ECC (Electronic Creative Clock) - Raspberry Pi System Document

================================================================================

本ドキュメントは、複合創造実験にて製作中の「枕型目覚ましデバイス（ECC）」のRaspberry Pi側制御ソフトウェアに関する詳細を記述したものです。

このデバイスは、単なる時間によるアラームではなく、枕に内蔵された圧力（重量）センサによって「ユーザーが実際に枕から頭を離したか」をリアルタイムで検知し、確実な起床を促すインテリジェントな目覚ましシステムを目指しています。

実装環境・ハードウェア仕様の詳細

■ ターゲットデバイス

Raspberry Pi (モデル 4B / 5 推奨)

マルチスレッド処理とGUI描画を並行して行うため、一定以上の処理能力を持つモデルを推奨します。

OS: Raspberry Pi OS (64-bit版 Bookworm以降を推奨)

■ ハードウェア構成とインターフェース (Hardware Interface Specs)

重量センサ (圧力センサ) 回路:

ADCチップ: MCP3208 (12bit, 8ch シリアル入力ADC) を採用。

接続方式: 高速なサンプリングを実現するため、ハードウェアSPI通信を利用します。

デバイスパス: /dev/spidev0.0

ADC入力: チャネル0にセンサ値を入力。12bit（0〜4095）の分解能で、微細な荷重変化を検知します。

アラーム出力ユニット (アクチュエータ):

制御方式: ハードウェアPWM (Pulse Width Modulation) を使用。

接続パス: /sys/class/pwm/pwmchip0/pwm0/

利点: CPU負荷を抑えつつ、安定した周波数でブザーやモーターを制御可能です。極性(polarity)や周期(period)の動的な変更に対応しています。

表示・操作UI (Graphical User Interface):

描画フレームワーク: SFML (Simple and Fast Multimedia Library) を使用。

解像度: 1280x720 (HD) を基準に設計。

表示形態: 枕元での視認性を考慮したフルスクリーン表示。

入力制御: アプリケーションの安全な終了にはキーボードの「Esc」キーを使用します。

依存関係とライブラリの役割 (Dependencies)

本システムは、モダンなC++17の機能を活用しており、以下の外部ライブラリに依存しています。

SFML 3.0.0 (Graphics/Window/System):

時計の文字盤や設定画面の描画、およびユーザー入力を一括して管理します。CMakeのFetchContent機能により、ビルド時に自動的に取得・ビルドされます。

csv-parser (by vincentlaucsb):

アラーム時刻の設定データを管理するCSVファイルの読み込みに使用します。型安全かつ高速なパースが可能で、複雑なスケジュール管理を容易にします。

libgpiodcxx (C++ binding for Linux GPIO):

従来のsysfs経由の制御に代わり、最新のキャラクタデバイス経由でGPIOを操作するための標準ライブラリです。

OpenGL / Mesa / FreeGLU:

SFMLがハードウェアアクセラレーションを用いて描画を行うためのグラフィックス基盤です。

セットアップ・環境構築手順

■ システム依存パッケージの一括インストール
ビルド環境を整えるために、まず以下のシェルスクリプトを実行してください。このスクリプトは、必要なグラフィックスライブラリやビルドツールを自動で揃えます。

# 実行権限を付与
chmod +x install/sfml-setup.sh
# セットアップスクリプトの実行
./install/sfml-setup.sh


■ GPIO制御ライブラリの個別インストール
libgpiodのC++バインディングはシステム側に必要です。以下のコマンドでインストールしてください。

sudo apt update
sudo apt install libgpiod-dev libgpiodcxx-dev


詳細なビルド方法 (Build Process)

本プロジェクトはCMakeを利用したアウトオブソースビルドを推奨しています。ソースディレクトリを汚さずにビルドを行うため、以下の手順に従ってください。

# Raspiディレクトリに移動
cd Raspi

# ビルド専用ディレクトリを作成して移動
mkdir -p build && cd build

# CMakeによる構成 (依存ライブラリのダウンロードが含まれるため、初回は時間がかかります)
cmake ..

# 並列ビルドを実行 (コア数に合わせて最適化されます)
make -j$(nproc)


ビルドが完了すると、build ディレクトリ内に実行バイナリ ECC_device が出力されます。

デプロイと運用方法

■ アプリケーションの実行
SPIデバイスやPWMクラスへのアクセスには特権が必要です。基本的には sudo を付与して実行します。

sudo ./ECC_device


■ データ永続化とディレクトリ仕様
本システムはLinuxの標準的な作法に従い、アプリケーションデータを以下のディレクトリで管理します。

保存先ディレクトリ: /var/lib/ECC_device/

主要データファイル: alarm_list.csv (アラーム設定の履歴やスケジュール)

プログラム起動時にこのディレクトリが存在しない場合、filesystem_helper が自動的に作成を試みます。手動で設定を編集する場合は、このCSVファイルを直接編集することも可能です。

実装の内部構造 (Thread Architecture)

システム全体の応答性を維持し、センサの取りこぼしを防ぐため、以下の4つのスレッドが並列で動作しています。各スレッド間のフラグ管理には std::atomic を使用し、スレッドセーフな設計を実現しています。

メインスレッド (GUI & Event Loop):
SFMLのウィンドウ制御を担当。描画更新（FPS管理）と、Escキーなどのユーザー入力を監視します。

時計システムスレッド (Clock Service):
std::chrono を用いて、ナノ秒精度のシステムクロックから現在時刻を常に更新し、他スレッドが参照できる共有メモリへ書き込みます。

アラーム監視スレッド (Alarm Logic):
CSVから読み込まれたアラームリストと現在時刻を常に照合します。条件が一致した際、アラームフラグを「鳴動状態」に切り替えます。

ハードウェア制御スレッド (Hardware I/O):
SPI経由で重量センサの電圧値をサンプリングし、独自のアルゴリズムで「起床」を判定します。また、判定結果に基づきPWM出力を制御して物理的なアラーム動作を行います。
